# LabelHub 单元测试报告

生成日期：2026-06-09  
测试目标：依据《LabelHub 数据标注平台 AI 全栈课题实现要求》中“关键流程有单元测试 / 集成测试”的验收要求，将当前测试体系中的纯函数、契约、配置解析、提示词解析和前端视图辅助逻辑整理为独立单元测试，并验证可单独运行。

## 一、测试边界

单元测试只验证小范围、低副作用逻辑，不依赖真实 MySQL、浏览器页面、真实 LLM 网络请求或后台常驻进程。需要跨 API、数据库、状态机、文件生成或 worker 调度的内容统一放入集成测试。

当前单元测试共 78 个用例：

| 模块 | 测试入口 | 用例数 | 说明 |
| --- | --- | ---: | --- |
| 前端 Web | `pnpm --filter @labelhub/web test:unit` | 54 | 路由、视图格式化、模板搭建 helper、导出/审核/数据集页面辅助逻辑 |
| 后端 API | `cd apps/api && uv run pytest tests/unit` | 15 | 鉴权契约、OpenAPI/Entity/迁移契约、LLM Client 配置与错误映射 |
| Agent | `cd apps/agent && uv run pytest tests/unit` | 9 | AI 预审 DTO、OpenAI 兼容配置、Prompt 组装、结构化输出解析 |

## 二、测试结构

| 目录/命名 | 定位 |
| --- | --- |
| `apps/web/src/**/*.unit.test.ts(x)` | 前端单元测试，使用 `apps/web/vitest.unit.config.ts` 独立收集 |
| `apps/api/tests/unit/` | 后端单元与契约测试 |
| `apps/agent/tests/unit/` | Agent 单元与契约测试 |

## 五、结论

单元测试已经从集成测试中拆分出来，并能独立运行。当前单元测试覆盖了核心接口契约、前端视图纯逻辑、模板搭建核心 helper、OpenAI 兼容配置、Agent Prompt 与结构化解析等基础能力。

本次单元测试未暴露业务代码缺陷。后续新增功能时，应优先在对应模块补充 `*.unit.test.ts(x)` 或 `tests/unit/` 用例，再进入跨模块集成测试。
