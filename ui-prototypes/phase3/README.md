# Phase 3 标注工作流原型说明

本目录用于沉淀阶段 3 的目标态 UI 原型。原型以官方要求文档的 4.3「标注员工作台」为准，结合当前已实现的任务广场与标注工作台代码重新设计，不直接复制当前线上页面。

## 页面清单

| 页面 | 原型路径 | 对应正式路由 | 设计目标 |
| --- | --- | --- | --- |
| 任务广场 | `marketplace/code.html` | `/labeler/marketplace` | 帮助 Labeler 快速判断可领取任务、继续未完成题目、理解模板与质检要求 |
| 标注工作台 | `workspace/code.html` | `/labeler/assignments/:assignmentId` | 支持题目导航、动态表单作答、自动保存、LLM 辅助、校验与提交 |
| 我的贡献 | `contributions/code.html` | `/labeler/contributions` | 汇总已提交、待复审、已通过、被打回和草稿数据，支持快速回到处理现场 |
| 返修处理 | `revise/code.html` | `/labeler/assignments/:assignmentId/revise` | 展示审核意见、差异对比、返修表单、重新提交前检查 |

## 需求覆盖

- 任务市场：覆盖任务列表、任务状态、奖励/配额/模板信息、领取与继续作答入口。
- 标注页动态渲染：覆盖展示项、单行、多行、单选、多选、标签、富文本、LLM 建议组件等阶段 2 产出的运行时组件。
- 作答体验：覆盖上一题、下一题、跳过、报告题目、保存草稿、提交本题、自动保存状态和快捷键提示。
- 质量反馈：覆盖 AI 初筛、人审打回、返修原因、历史记录、错误提示和提交前检查。
- 我的数据：覆盖贡献统计、题目列表、状态筛选和继续处理入口。

## 设计原则

- 左侧导航承载一级工作域，右侧页面只承载当前任务流，避免入口重复造成路径混乱。
- 任务广场面向「选择与领取」，标注工作台面向「高效率作答」，我的贡献面向「结果追踪」，返修页面向「问题修复」。
- 标注工作台默认进入聚焦模式：最左侧全局导航收起为窄栏，工作台内部保留题目导航、作答区和上下文侧栏，优先保证主作答区宽度。
- 视觉风格保持企业内部平台的克制、密集和可扫描：低饱和背景、8px 卡片半径、清晰表格层级、重点操作使用蓝色主按钮。
- 所有原型按 1280×800 与大屏宽度做过浏览器检查，避免出现横向滚动和关键内容遮挡。

## 与当前实现的关系

- `marketplace` 与 `workspace` 是对当前已实现页面的目标态重设计，后续正式代码可以逐步按此原型收敛。
- `contributions` 与 `revise` 是阶段 3 后续能力的完整目标态，用于指导后续开发粒度落地。
- 原型只作为 UI/产品交互基准，不改变当前业务代码、接口和数据库结构。
