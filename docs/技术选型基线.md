# LabelHub 技术选型基线

## 1. 文档目的

本文档记录 LabelHub 当前已经确定的技术选型。后续实现必须以本文档为准；如需变更，需要先更新本文档、前后端 SDD 和系统架构文档，再进入代码实现。

## 2. 已确定选型

| 层 | 已确定选型 |
| --- | --- |
| 前端框架 | React 18 + TypeScript |
| UI 组件库 | Ant Design |
| 表单内核 | Formily + Schema 渲染 |
| 拖拽 | @dnd-kit/core |
| 状态管理 | Zustand |
| 后端 | Python |
| 数据库 | MySQL |
| LLM 接入 | OpenAI API 格式；Agent 默认 `BASE_URL=https://token-plan-cn.xiaomimimo.com/v1`、`MODEL_NAME=mimo-v2.5-pro`、thinking 关闭 |

## 3. 阶段 0 已确认补充选型

| 层 | 阶段 0 选型 |
| --- | --- |
| 前端构建 | Vite |
| 前端测试 | Vitest |
| 前端代码检查 | ESLint |
| 后端 Web 框架 | FastAPI |
| 后端数据校验 | Pydantic v2 |
| 后端 ORM | SQLAlchemy 2 |
| 后端迁移 | Alembic |
| MySQL Driver | PyMySQL |
| 鉴权方式 | HttpOnly Cookie Session |

## 4. 执行规则

- 前端页面、组件和状态管理不得引入与 Ant Design、Formily、`@dnd-kit/core`、Zustand 冲突的替代方案。
- 后端和 Agent 必须使用 Python，并通过 `uv` 管理包和虚拟环境。
- 数据库以 MySQL 为准，数据库设计和迁移不得默认面向其他数据库特性。
- LLM 接入层必须按 OpenAI API 格式封装；Agent 当前默认使用 `BASE_URL`、`MODEL_NAME` 和 `OPENAI_API_KEY` 环境变量，真实密钥只能放在本地 `.env` 或部署密钥中。当前 MiMo OpenAI 兼容服务关闭 thinking 需要在请求体中携带 `chat_template_kwargs.enable_thinking=false`。
- 队列实现尚未固定；确认时必须先更新 SDD。

## 5. 与 SDD 的关系

正式开发前必须完成以下对齐：

- `apps/web/SDD.md` 中的前端 VO、页面状态和接口调用必须符合本技术选型。
- `apps/api/SDD.md` 中的 Request、DTO、BO、Entity、VO 必须符合 Python + MySQL + OpenAI API 格式的实现边界。
- 前端 VO 字段名必须与后端返回 JSON 字段名一一对应。
- 后端 LLM 输出 DTO 必须先通过结构化模型校验，再进入审核业务流程。
