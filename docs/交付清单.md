# LabelHub 交付清单

本文档用于说明 `submission/` 目录中的交付材料组织方式。所有材料均基于当前代码实现整理，面向项目功能、架构、部署、测试和 AI Coding 过程追溯。

## 目录结构

```text
submission/
  README.md                         # 项目总览、启动方式、核心能力和交付说明
  交付清单.md                       # 当前交付目录结构和材料说明
  API文档.md                        # 后端 API 契约与请求/响应说明
  基础技术文档/
    需求分析文档.md
    Demo范围文档.md
    系统架构文档.md
    数据库设计文档.md
    技术选型基线.md
    单元测试报告.md
    集成测试报告.md
    真实浏览器全流程验收测试报告.md
  AI_Coding过程记录/
    AI Coding过程实现思路说明.md
    开发粒度与实施计划.md
    前端SDD.md
    后端SDD.md
  演示环境说明/
    生产部署说明.md
    演示环境说明.md
  Demo截图/
    README.md
    Demo截图说明.md
    *.png / *.jpeg
```

## 材料说明

| 分类 | 说明 |
| --- | --- |
| README 与交付清单 | 介绍 LabelHub 的业务目标、角色链路、技术栈、启动方式、测试方式、部署入口和交付目录结构 |
| API 文档 | 位于 `submission/API文档.md`，覆盖后端接口契约、请求参数、响应结构和典型调用链路 |
| 基础技术文档 | 覆盖需求、Demo 范围、系统架构、数据库设计、技术选型和测试报告 |
| AI Coding 过程记录 | 保留 SDD 契约、开发粒度与实施计划，以及 AI 辅助开发过程方法说明 |
| 演示环境说明 | 说明生产部署方式、演示环境访问地址、账号和推荐演示路径 |
| Demo 截图 | 收录基于当前真实页面重新截取的关键页面截图，便于快速了解产品形态 |

## 交付状态

当前版本已完成 Owner、Labeler、AI Agent、Reviewer 和数据导出五条主链路的闭环实现，并完成单元测试、集成测试与真实浏览器全流程验收记录。演示视频不在当前目录中收录，可在需要时基于演示环境说明录制。
