生成日期：2026-06-09  
测试目标：验证任务创建、数据导入、模板发布、标注作答、题目级 LLM 辅助、AI 预审、人工审核流转、数据验收和多格式导出等关键流程。

## 一、测试边界

集成测试用于验证多个模块之间的契约和状态流转，包括 API + 数据库会话、状态机、审计日志、文件生成、Agent worker 与模板运行时渲染。真实浏览器端到端操作验收见 `docs/真实浏览器全流程验收测试报告.md`。

当前集成测试共 75 个用例：

| 模块 | 测试入口 | 用例数 | 说明 |
| --- | --- | ---: | --- |
| 前端 Web | `pnpm --filter @labelhub/web test:integration` | 6 | 模板运行时、Renderer、条件显示、提交值校验与隐藏字段裁剪 |
| 后端 API | `cd apps/api && uv run pytest tests/integration` | 67 | Owner、Labeler、Reviewer、AI 预审、导出全链路 |
| Agent | `cd apps/agent && uv run pytest tests/integration` | 2 | worker 成功写回与失败写回 |
